# Инструменты глубокого исследования (DeepResearch)

**OpenAI DeepResearch** — новый режим в ChatGPT, предназначенный для комплексных научных обзоров. По описанию, DeepResearch «автономно проводит всесторонние интернет-исследования по сложным темам», обрабатывая сотни источников (текст, изображения, PDF) и выдавая подробные аналитические отчёты【6†L36-L43】【6†L41-L47】. Система основана на расширенной версии модели (O3) и использует браузер и Python-инструменты для поиска. В ходе работы DeepResearch показывает источники и промежуточные результаты, а итоговый ответ содержит структурированный отчёт с возможностью встраивать изображения и графики【6†L95-L104】. Режим доступен сейчас только пользователям профессиональной подписки (скоро планируется Plus/Team). **Возможности:** глубокий обзор литературы, «сжатие» часов работы в минуты, максимальная прозрачность (все источники документируются)【6†L36-L43】【6†L77-L83】. **Ограничения:** требуется подписка, длительное время (5–30 минут на задачу) и доступ в Интернет; пока ориентирован на англоязычные запросы.

**Perplexity Deep Research** — специализированный режим в Perplexity.ai для научных исследований. По словам автора блога, этот инструмент «ставит точность и достоверность на первое место»: он не просто ищет совпадения ключевых слов, а анализирует информацию из научных статей, новостных сайтов, форумов и т.д., выявляя противоречия и синтезируя результаты в структурированный отчёт【7†L68-L72】【8†L155-L163】. Интерфейс прост: задаётся запрос и Perplexity возвращает связный текст с выводами и ссылками на источники【8†L177-L185】. Есть бесплатный лимит (ок. 5 запросов/день) и платный Pro ($20/мес, 500 запросов)【8†L157-L164】. **Преимущества:** бесплатен для базового использования, интегрирован поиск по миллионам источников, выводит ссылки. **Недостатки:** ограниченное число запросов в бесплатной версии, качество синтеза зависит от формулировки промпта【8†L189-L197】, меньше наработок по пользовательским сценариям в сравнении с ChatGPT.  

## Claude Code

**Claude Code** — агентная система автоматизации программирования от Anthropic. Он «читает ваш кодовый репозиторий, вносит изменения в файлы, запускает тесты и оформляет коммиты»【10†L14-L18】【11†L79-L87】. В отличие от автодополнения, Claude Code действует на уровне всего проекта: понимает всю кодовую базу и может одновременно править несколько файлов, добавлять новые функции и выполнять рефакторинг. Типичные задачи включают: поиск по не знакомому коду и объяснение архитектуры, добавление фич «по описанию», исправление ошибок тестов и CI-фейлов, массовые операции (например, правка стиля по всему проекту)【10†L42-L49】【10†L72-L80】. Claude Code доступен в виде CLI, плагинов для VS Code, JetBrains IDE и настольного/веб-приложения【11†L93-L101】【11†L107-L115】. **Возможности:** работает с любыми инструментами разработчиков (git, CLI), разбирает зависимости, интегрируется с CI/CD (GitHub Actions, GitLab)【10†L72-L80】. Подходит как разработчикам, так и неинженерам (предприниматели, продуктовые менеджеры), желающим описать задачу на естественном языке【10†L14-L18】【10†L78-L82】. **Ограничения:** требуется подписка Claude; инструмент заточен под код, а не задание научной темы; безопасность реализована через режимы контроля (при необходимости просит подтверждения перед изменением файлов)【10†L114-L122】【10†L126-L134】. 

## AutoResearch (Karpathy)

**AutoResearch** — открытый проект Андрея Карпати для автопроведения ML-экспериментов. Это Python-фреймворк, где **агент (LLM)** сам генерирует и прогоняет варианты экспериментов. Суть: три файла (*prepare.py* – оценка, *train.py* – модифицируемый код, *program.md* – инструкции) задают контракт работы【17†L615-L623】. Агент последовательно «предлагает → тренирует (5 мин на эксперимент) → оценивает» модификации модели и оставляет только те изменения, которые улучшают метрику (валидационная loss)【17†L549-L552】. Так можно за ночь получить сотни экспериментов на одной GPU с фиксированным временным бюджетом. **Возможности:** полностью автоматизированное улучшение нейронной сети (например, NanoGPT) без ручного подбора гиперпараметров. Система свободна (MIT-лицензия), и, как сообщают, уже показала улучшения в различных задачах. **Ограничения:** фокусируется на ML-экспериментах (в первую очередь языковые модели), требует GPU и базовой настройки. Не предназначен для автоматизации научного письма: генерирует код для тренировок, но отчёты и анализ результатов — на доле человека.  

## AutoResearchClaw

**AutoResearchClaw** — проект (aiming-lab/Sakana) для полного цикла исследований «от идеи до статьи». По описанию, при вводе темы система возвращает *«полную академическую статью с реальной литературой из OpenAlex, Semantic Scholar и arXiv»*, включая сгенерированный код экспериментов, результаты, графики и LaTeX-шаблон для NeurIPS/ICML/ICLR【28†L399-L405】. Конвейер включает 23 этапа: разбиение задачи, поиск литературы, генерация гипотез, проектирование экспериментов, выполнение и анализ опытов, написание черновика статьи и рецензия【26†L598-L606】【26†L608-L617】. Поддерживаются полностью автономный режим и **Co-Pilot**-режим с участием человека на ключевых этапах (выбор гипотез, проверка дизайна, правка текста)【28†L399-L405】【26†L482-L490】. Другие особенности: многоагентные дебаты при формулировке гипотез и рецензировании, механизм «pivot/refine» (автоперенаправление исследования при необходимости), проверка фактов и ссылок, самосовершенствование на предыдущих шагах【26†L482-L490】【28†L399-L405】. Инструмент доступен как CLI (устанавливается через `pip`), может работать в контейнере Docker, а также интегрируется с платформой OpenClaw и любыми AI-агентами через протокол ACP【28†L427-L434】【26†L507-L516】. **Преимущества:** обещает **«одна команда — одна статья»**, автогенерируя полный пакет артефактов (Markdown, LaTeX, BibTeX, код экспериментов, графики)【28†L399-L405】【28†L407-L415】. **Недостатки:** очень сложен в настройке; требует значительных вычислительных ресурсов (машина с GPU/MPS/CPU для экспериментов), знаний работы с AI и Docker. Проект сравнительно новый (релиз первой версии март 2026) и пока скорее исследовательский, чем «промышленно зрелый».  

## Другие современные инструменты

- **AI Scientist (Sakana/MetaClaw)** – исследовательская система, которая «автоматизирует весь жизненный цикл исследований»: генерирует идеи, пишет код, проводит эксперименты, визуализирует результаты и пишет полный научный доклад【30†L34-L39】. В 2025 г. она провела исследования в области ML, выведя новые модели (например, для диффузионных моделей и трансформеров). Система даже включает модуль автоматической рецензии, что позволяет ей итеративно улучшать свои «открытия»【30†L34-L39】. Этот проект показал, что современным LLM по силам практически полностью автоматизировать исследования на уровне качества конференций (хотя ещё встречаются ошибки).  

- **AI-Researcher (HKUDS, NeurIPS 2025)** — похожая инициатива, рассчитанная на простоту использования. Достаточно дать системе список релевантных статей, а она сама спланирует и проведёт исследования, сохраняя историю изменений и экспериментов【32†L1245-L1248】. Разработчики подчёркивают, что пользователю не нужно писать код или настраивать сложную среду — весь процесс автоматизирован «из коробки»【32†L1245-L1248】.  

- **Elicit** – веб-сервис для научного поиска и анализа данных, ориентированный на исследования и обзоры. По сути, это AI-помощник для литературы: вводится научный вопрос, а система ищет по базе (~140 млн публикаций), извлекает ключевую информацию из релевантных статей и формирует структурированный отчёт с цитатами【33†L37-L40】. Elicit особенно полезен для систематических обзоров: он может автоматически скринировать сотни статей, извлекать данные (напр. статистику, выборки) и экономит до 80% времени по сравнению с ручной работой. **Особенности:** детализированный отчёт по запросу, поддержка интерактивного выбора и уточнения результатов, интегрированный менеджер библиографии, уведомления об обновлениях литературы.  

- **PaperGuide AI** – коммерческая платформа «всё в одном» для исследований. Она включает AI-поиск по научным работам, генератор обзоров, а также AI-редактор текста. По описанию, PaperGuide может «создавать полные черновики с правильными ссылками за минуты» и поддерживает работу с базой 200+ млн статей【35†L112-L113】. Пользователь может загружать PDF, вести диалоги с содержимым статьи, а AI поможет с рефератами, оформлением цитат и т.д. Интерфейс напоминает комбинацию научного поисковика и инструмента для написания с искусственным интеллектом.  

- **Другие заметные инструменты:** GitHub Copilot (AI для автодополнения кода), OpenAI’s S4 Science Companion (для поиска и написания), «Galactica» и ее аналоги для поиска по науке (хотя Galactica уже закрыта). Библиографические менеджеры нового поколения (Scite.ai, ConnectedPapers, ResearchRabbit) тоже используют AI для навигации по публикациям, но они больше помогают исследователю, а не генерируют текст.  

## Сравнение и рекомендации

- **Наполнение и сфера:** DeepResearch- и Perplexity-режимы (ChatGPT, PaperGuide, Elicit) фокусируются на сборе и анализе литературных данных – они идеальны для обзоров, генерации идей и формулировки вопросов. Claude Code специализируется на кодинге, а не на научном тексте. Autoresearch ориентирован на задачи машинного обучения (подбор параметров моделей), AutoResearchClaw/AI Scientist – на полный цикл исследований (идея → код → эксперименты → статья).  
- **Удобство:** Для обработки запросов в естественном языке (поиск ответов, обобщение знаний) проще всего использовать веб-сервисы (DeepResearch ChatGPT, Perplexity, Elicit, PaperGuide) — они не требуют установки. Для разработки кода нужен Claude Code CLI или IDE-плагин. Автосистемы типа AutoResearch и AutoResearchClaw требуют настройки окружения (Python, GPU, Docker) и более технической подготовки.  
- **Интеграция:** Claude Code хорошо вписывается в существующие рабочие процессы разработки (VS Code, git, CI). AutoResearchClaw интегрируется с платформой OpenClaw и может управляться через чат-боты (Discord, Telegram) или напрямую CLI. DeepResearch встроен в ChatGPT-окно, Perplexity – отдельный сайт.  
- **Ограничения:** Большинство перечисленных систем не заменяют исследователя полностью. Например, они могут генерировать правдоподобные идеи и тексты, но без контроля человека иногда возникают ошибки или несоответствия. Цитатные сервисы (DeepResearch, Elicit, PaperGuide) доверяют проверенным базам данных, однако критично всегда самостоятельно верифицировать источники. Автономные «агенты» (AutoResearch, AutoResearchClaw, AI Scientist) могут требовать значительных ресурсов и иногда делать нелогичные шаги, поэтому рекомендуется следить за промежуточными результатами и при необходимости корректировать стратегию.  

**Рекомендации:** Для обзора литературы и генерации идей стоит попробовать DeepResearch (ChatGPT Pro) и Perplexity DeepResearch, а также Elicit – они быстро дают контекст и ссылки【6†L36-L43】【33†L37-L40】. Для написания кода к научному проекту подойдут Claude Code или GitHub Copilot (особенно при наличии большого кода в репозитории)【10†L14-L18】【11†L79-L87】. Для экспериментов ML-практики — Autoresearch от Карпати (если есть GPU)【17†L549-L552】. Для самых амбициозных задач по созданию полного исследования из идеи можно изучить AutoResearchClaw или AI Scientist, но они требуют серьезной подготовки и пока экспериментальны【28†L399-L405】【30†L34-L39】. Также имеет смысл следить за новыми разработками (например, AI-Researcher) и использовать сочетание инструментов: например, сначала собрать литературу через Elicit/DeepResearch, затем сгенерировать код через Claude Code, а итоговый черновик формализовать с помощью PaperGuide или ChatGPT с включением найденных ссылок【35†L112-L113】【33†L37-L40】. 

**Источники:** официальные описания и документация инструментов【6†L36-L43】【10†L14-L18】【17†L549-L552】【20†L378-L381】, обзоры и статьи в СМИ【7†L68-L72】【30†L34-L39】, а также сайты разработчиков (Claude Docs, Perplexity.ai, Paperguide.ai, Elicit.com)【11†L79-L87】【35†L112-L113】. Эти ресурсы подтверждают возможности и ограничения описанных систем.
